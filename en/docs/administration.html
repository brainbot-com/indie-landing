Users, Dashboard & Updates

This page covers the operational steps that matter after the first successful use: password changes, user creation, dashboard checks and controlled updates.

Create users and change the password

First change the initial password from the handout. Then create the actual user accounts for the people who will work on the system. Keep admin rights limited.

Dashboard and vital signs

The dashboard is the first place to check service status, coarse resource use and the general health of the system. For day one, a pragmatic overview is enough.

Place updates in the right order

Updates should be deliberate. Right after first setup, confirming stability is often more useful than updating immediately.
