Shell & Expert Access

The shell is not a first-run feature. It is a tool for diagnostics, logs, deeper administration and recovery work once the browser-based path has already been validated.

When this page becomes relevant

If browser access, OpenWebUI and the dashboard are not enough to understand a problem, shell access becomes relevant.

What this page intentionally does not do

This documentation does not encourage destructive commands or shortcuts. Shell access is an expert operational tool, not a replacement for the normal path.

What a deeper expert page should cover later

  • How shell access is reached
  • Logs and service status
  • Container-related diagnostics
  • Persistent data and recovery paths
