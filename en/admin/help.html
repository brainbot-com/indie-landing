Indiebox Admin

Short explanations for the core admin workflows.

Notifications

Under Notifications you manage email templates that go out automatically on status changes.

Triggers

The only trigger today is order.paid — fired when Mollie confirms a payment. All active templates matching this trigger (and the order's language filter) are sent.

Template parts

  • Name — internal label, admin only.
  • Description — short note about the purpose.
  • Trigger — the event it is bound to.
  • Recipient — Admin (uses the configured ORDER_NOTIFICATION_TO), Customer (email from the order) or Custom address.
  • Language filter — if set, template only fires for orders in that language. "Any" ignores the order's language.
  • Subject — the email subject, may contain tokens.
  • Plain text version — fallback for clients without HTML support.
  • HTML version — the visual email.
  • Enabled — disabled templates are skipped.

Tokens

Placeholders in the form {{path.to.value}} are replaced with real order data on send. Values inside HTML are automatically HTML-escaped. Keys ending in Html (e.g. {{billingAddress.blockHtml}}) are inserted as raw HTML fragments.

The full token list is available inside the editor under "Available tokens" — click to copy.

Editing HTML and Preview

Toggle between Edit and Preview at the top of the HTML section. Preview renders the template with sample data so you can see what the recipient will get.

Images

Images are stored content-addressed under /mail/<hash>.<ext> and are publicly accessible. Upload in the editor, then paste the URL into the HTML as <img src="/mail/…">.

At send time the relative URL is rewritten to the full domain, so the same template works unchanged on staging and production.

Allowed formats: PNG, JPEG, GIF, WebP; max 3 MB per file.

Import from an external editor

Tools like Stripo or BeeFree export ready-made HTML emails. In the HTML section choose Import HTML, paste the exported code — external image references are downloaded automatically, stored in /mail/, and the HTML is rewritten accordingly. Afterwards add tokens like {{customer.firstName}} where needed.

Test send

The Send test button ships the saved version with sample order data to the address you pick. The subject line is prefixed with [TEST].

Transport staging → live

Finish and test on staging, then click Export template — a .json file downloads that contains the template and all used images.

On live, click Import template in the card header, pick the file — template and images land in place. Relative image URLs stay valid across environments.

Mailgun configuration

For mails to actually go out, the runtime env on the server needs the following variables:

MAILGUN_API_KEY=…
MAILGUN_DOMAIN=mg.indiebox.ai
MAILGUN_REGION=eu
ORDER_NOTIFICATION_FROM=Indiebox <indiebox@brainbot.com>
ORDER_NOTIFICATION_TO=indiebox@brainbot.com

Without the first three, the backend silently skips sending — but templates can still be edited and previewed.

Inventory

Under Inventory you manage device models, supplier orders, and individual devices. The full Indiebox workflow (ordered → in stock → prepared → shipped → installed) is tracked here.

Orders

Under Orders you see all incoming customer orders with payment status and fulfillment progress. A paid order can be assigned to a specific device from inventory and walked through the shipping and installation steps.

Users

Under Users you manage admin accounts and roles. Only accounts with the admin role can see internal admin areas.
