Kurze Erklärungen zu den wichtigsten Abläufen im Admin-Bereich.

Benachrichtigungen

Auf der Seite Benachrichtigungen verwaltest du E-Mail-Templates, die automatisch bei Statusänderungen versendet werden.

Auslöser

Aktuell gibt es einen Auslöser: order.paid — wenn eine Bestellung von Mollie als bezahlt bestätigt wird. Zu diesem Auslöser laufen alle aktiven Templates mit passendem Sprachfilter.

Aufbau eines Templates

  • Name — interne Bezeichnung, nur im Admin sichtbar.
  • Beschreibung — kurze Notiz, wofür das Template genutzt wird.
  • Auslöser — an welches Ereignis es gebunden ist.
  • Empfänger — Admin (interne Mail an die konfigurierte ORDER_NOTIFICATION_TO-Adresse), Kunde (Adresse aus der Bestellung) oder feste Adresse.
  • Sprachfilter — wenn gesetzt, läuft das Template nur bei Bestellungen mit genau dieser Sprache. „Alle" ignoriert die Sprache.
  • Betreff — Subject-Zeile, darf Tokens enthalten.
  • Plain-Text-Version — fällt für Mail-Clients ohne HTML-Support zurück.
  • HTML-Version — die eigentliche visuelle E-Mail.
  • Aktiv — deaktivierte Templates werden beim Auslöser übersprungen.

Tokens

Platzhalter im Format {{pfad.zum.wert}} werden beim Versand durch echte Bestelldaten ersetzt. Im HTML werden Werte automatisch HTML-escaped. Keys mit der Endung Html (z. B. {{billingAddress.blockHtml}}) werden als fertiges HTML-Fragment eingesetzt.

Die vollständige Tokenliste findest du im Editor unter „Verfügbare Tokens" — mit Klick kopierbar.

HTML bearbeiten und Vorschau

Im HTML-Editor wechselst du oben rechts zwischen Bearbeiten und Vorschau. Die Vorschau rendert das Template mit Beispieldaten und zeigt, wie die Mail beim Empfänger ankommt.

Bilder

Bilder liegen content-adressiert unter /mail/<hash>.<ext> und sind öffentlich erreichbar. Im Editor lädst du sie hoch und bekommst eine relative URL, die du im HTML als <img src="/mail/…"> einfügst.

Beim Versand wird die relative URL automatisch zur vollen Domain erweitert — das gleiche Template funktioniert deshalb unverändert auf Staging und Live.

Erlaubt sind PNG, JPEG, GIF, WebP, maximal 3 MB pro Datei.

Template aus externem Editor importieren

Tools wie Stripo oder BeeFree exportieren fertige HTML-Mails. Wähle im HTML-Bereich HTML importieren, füge den Export-Code ein, bestätige — externe Bildreferenzen werden automatisch heruntergeladen, in /mail/ abgelegt und der HTML-Code entsprechend umgeschrieben. Danach Tokens wie {{customer.firstName}} an den gewünschten Stellen einsetzen.

Test-Versand

Der Button Test-Mail senden verschickt die aktuell gespeicherte Version mit Beispielbestelldaten an eine Adresse deiner Wahl. Der Betreff wird mit [TEST] vorangestellt.

Transport Staging → Live

Auf Staging fertigstellen und testen, dann Mail-Template exportieren — es lädt eine .json-Datei herunter, die das Template und alle verwendeten Bilder enthält.

Auf Live oben rechts in der Kartenleiste Mail-Template importieren, die Datei wählen — Template und Bilder werden übernommen. Relative Bild-URLs bleiben gültig.

Mailgun-Konfiguration

Damit Mails tatsächlich versendet werden, müssen auf dem Server in der Laufzeit-Env folgende Variablen gesetzt sein:

MAILGUN_API_KEY=…
MAILGUN_DOMAIN=mg.indiebox.ai
MAILGUN_REGION=eu
ORDER_NOTIFICATION_FROM=Indiebox <indiebox@brainbot.com>
ORDER_NOTIFICATION_TO=indiebox@brainbot.com

Fehlt eine der ersten drei, überspringt der Backend-Versand still — Templates lassen sich aber trotzdem bearbeiten und in der Vorschau prüfen.

Bestand

Unter Bestand werden Gerätemodelle, Lieferantenbestellungen und einzelne Geräte verwaltet. Jeder Indiebox-Workflow (bestellt → im Lager → bereitgestellt → versendet → installiert) wird hier nachvollzogen.

Bestellungen

Unter Bestellungen siehst du alle eingehenden Kundenbestellungen mit Zahlstatus und Fulfillment-Fortschritt. Eine bezahlte Bestellung lässt sich einem konkreten Gerät aus dem Bestand zuordnen und durch die Versand- und Installationsschritte führen.

Benutzer

Unter Benutzer legst du Admin-Konten an und verwaltest Rollen. Nur Konten mit der Rolle admin sehen die internen Admin-Bereiche.
