Nutzer, Dashboard & Updates

Diese Seite erklärt die Schritte, die nach dem ersten erfolgreichen Start wichtig werden: Passwort ändern, Nutzer anlegen, Dashboard prüfen und Updates einordnen.

Nutzer anlegen und Passwort ändern

Ändere zuerst das Passwort aus dem Handout. Lege danach die Benutzerkonten für die Personen an, die tatsächlich mit dem System arbeiten sollen. Admin-Rechte sollten nur dort vergeben werden, wo sie wirklich gebraucht werden.

Dashboard und Vitaldaten

Das Dashboard ist die erste Anlaufstelle für den Zustand des Systems. Prüfe laufende Dienste, grobe Ressourcennutzung und den allgemeinen Status. Für den ersten Tag reicht ein kurzer Überblick.

Updates richtig einordnen

Updates sollten kontrolliert erfolgen. Nach dem ersten Start ist es oft sinnvoller, den aktuellen Zustand kurz zu prüfen, statt sofort zu aktualisieren. Wichtig ist, dass das System stabil läuft und der Ausgangszustand bekannt ist.

Nicht sofort aktualisieren

Ein verfügbares Update muss nicht automatisch sofort eingespielt werden. Zuerst prüfen, ob das System stabil läuft.
